PROJECT CONTEXT & TECH STACK (READ THIS FIRST)
────────────────────────────────────────────────
This is a 2D top-down medieval strategy game built with:
  - Phaser 3.80 with Arcade Physics (zero gravity, top-down movement)
  - Vite 5.4 (ES modules, "type": "module" in package.json)
  - simplex-noise for terrain generation
  - 1024×768 canvas, scaled to fit browser (Phaser.Scale.FIT)
  - WebGL with Canvas fallback, pixelArt: true, roundPixels: true
  - ALL art is procedurally drawn pixel art — NO external image files

SCENE FLOW (current):
  PreloadScene → BootScene → MainMenuScene → CharSelectScene → GameScene ↔ VillageScene
  HUDScene runs as parallel overlay on top of GameScene

SCENE FLOW (after build):
  PreloadScene → BootScene → MainMenuScene → LoginScene → CharSelectScene → GameScene ↔ VillageScene
  HUDScene runs as parallel overlay on top of GameScene

KEY ARCHITECTURE PATTERNS TO FOLLOW:
  - SpriteFactory.generateAll(scene) is called in PreloadScene.create() to
    generate ALL textures before any other scene runs. New textures go here.
  - SpriteFactory.registerAnimations(scene) is also called in PreloadScene.
    New animations go here.
  - Sprites are drawn in grayscale/white using Phaser Graphics, then tinted
    at runtime. Follow this pattern for ALL new sprites.
  - Systems are instantiated in GameScene.create() and stored as this.* properties
  - HUDScene receives references to player and systems via scene.launch() data parameter
  - Events flow from GameScene → HUDScene via gameScene.events.emit()
  - E key is the universal interaction key (village entry, NPC talk, shop)
  - UI uses Georgia serif font, colors from ParchmentColors.js UI_COLORS
  - All game constants live in src/constants.js

CRITICAL: Do NOT break existing features. The game has working combat, enemies,
inventory, hotbar, shop, minimap, territory capture, village scenes, weather
particles, resource collection. All of it must keep working after every phase.

STEP 5: Create server/src/server.js
────────────────────────────────────

  - Enable CORS for all origins (these are kids on various devices)
  - JSON body parser
  - Serve static files from ../dist if it exists (for production)

  REST endpoints:
    POST /api/auth/create   — body: { displayName, selectedClass }
                              → { player, playerCode, token }
    POST /api/auth/login    — body: { playerCode }
                              → { player, token }
    GET  /api/player/:id    — → player data (parse JSON fields)
    PATCH /api/player/:id   — body: partial player fields → updated player
    GET  /api/submissions   — → array of submissions with voter counts
    POST /api/submissions   — auth required, body: { content, category }
    POST /api/submissions/:id/vote — auth required
    GET  /api/changelog     — → array of changelog entries
    GET  /api/changelog/latest — → single latest entry
    GET  /api/monuments     — → array of monuments
    POST /api/admin/approve/:id  — requires x-admin-key header
    POST /api/admin/changelog    — requires x-admin-key header

  Auth middleware: check Authorization: Bearer <token> header, decode JWT,
  attach req.playerId. Apply to routes that need it.

  WebSocket server: attach to same HTTP server via ws library
    On connection:
      - Wait for first "join" message with { playerId, token }
      - Verify token, look up player
      - Add to gameState, broadcast "player_joined" to others
      - Send "world_state" with all current players + chatHistory to joiner
    On message (after join):
      "position" { x, y, facing, animation }
        → update gameState, broadcast "player_moved" to others
      "chat" { message }
        → validate (max 200 chars, rate limit 1/2sec), run word filter
        → add to chatHistory, broadcast "chat_message" to all
      "action" { actionType, data }
        → broadcast to others (for future extensibility)
    On close:
      → remove from gameState, broadcast "player_left"

STEP 1: Create src/network/NetworkManager.js
─────────────────────────────────────────────
  Singleton class. Store instance as static property.

  constructor(scene):
    - this.scene = scene
    - this.ws = null
    - this.connected = false
    - this.playerId = null
    - this.token = null
    - this.remotePlayers = new Map()   // playerId → RemotePlayer instance
    - this.positionSendTimer = 0
    - this.positionSendInterval = 100  // 10hz
    - this.reconnectAttempts = 0
    - this.maxReconnectAttempts = 5

  connect(playerId, token):
    - Determine WS URL: in dev use relative '/ws', in prod use location.host
    - Actually, use: const wsProtocol = location.protocol === 'https:' ? 'wss:' : 'ws:';
      const wsUrl = `${wsProtocol}//${location.host}/ws`;
    - Create WebSocket, set up onopen/onmessage/onclose/onerror
    - On open: send { type: 'join', payload: { playerId, token } }
    - On message: parse JSON, route by type to handler methods
    - On close: attempt reconnect with exponential backoff

  Handler methods:
    handleWorldState(payload):
      - payload.players is array of { playerId, displayName, selectedClass, x, y, facing }
      - For each, create a RemotePlayer (see below) and add to this.remotePlayers
      - payload.chatHistory → emit to chat UI

    handlePlayerJoined(payload):
      - Create new RemotePlayer, add to this.remotePlayers
      - Emit event for chat system message

    handlePlayerLeft(payload):
      - Get RemotePlayer from map, call .destroy(), delete from map

    handlePlayerMoved(payload):
      - Get RemotePlayer, call .setTargetPosition(x, y, facing, animation)

    handleChatMessage(payload):
      - Emit 'chatMessage' event on scene with payload data

  sendPosition(x, y, facing, animation):
    - Only if connected
    - Send { type: 'position', payload: { x, y, facing, animation } }

  sendChat(message):
    - Send { type: 'chat', payload: { message } }

  update(dt):
    - this.positionSendTimer += dt
    - If timer >= positionSendInterval, send current player position, reset timer
    - For each RemotePlayer in map: call .update(dt) for interpolation

  disconnect():
    - Close WebSocket, clear remotePlayers

  Static getInstance() / setInstance() for singleton access.

STEP 2: Create src/entities/RemotePlayer.js
────────────────────────────────────────────
  Similar to how Player creates its sprite, but controlled by network data.

  constructor(scene, playerId, displayName, selectedClass, x, y):
    - Import CHARACTER_CLASSES from '../entities/CharacterClasses.js'
    - this.scene = scene
    - this.playerId = playerId
    - this.displayName = displayName
    - this.classData = CHARACTER_CLASSES[selectedClass] || CHARACTER_CLASSES.Knight
    - Create sprite: scene.add.sprite(x, y, 'player_idle_down_0')
    - this.sprite.setTint(this.classData.color)
    - this.sprite.setDepth(10)
    - DO NOT add physics body — remote players don't need physics
    - Create name label: scene.add.text(x, y - 20, displayName, {
        fontSize: '10px', fontFamily: 'Georgia, serif', color: '#f4e4c1',
        stroke: '#2c1810', strokeThickness: 2 })
    - this.nameLabel.setOrigin(0.5).setDepth(11)
    - Interpolation state:
      this.targetX = x, this.targetY = y
      this.currentX = x, this.currentY = y
      this.facing = 'down', this.currentAnim = ''

  setTargetPosition(x, y, facing, animation):
    - this.targetX = x, this.targetY = y
    - this.facing = facing || this.facing
    - Update animation based on whether position is changing

  update(dt):
    - Lerp currentX/Y toward targetX/Y:
      const lerpSpeed = 0.15;
      this.currentX += (this.targetX - this.currentX) * lerpSpeed;
      this.currentY += (this.targetY - this.currentY) * lerpSpeed;
    - this.sprite.setPosition(this.currentX, this.currentY)
    - this.nameLabel.setPosition(this.currentX, this.currentY - 20)
    - Play appropriate animation:
      const isMoving = Math.abs(this.targetX - this.currentX) > 0.5 ||
                       Math.abs(this.targetY - this.currentY) > 0.5;
      const animKey = isMoving ? `player_walk_${this.facing}` : `player_idle_${this.facing}`;
      if (this.currentAnim !== animKey) {
        this.currentAnim = animKey;
        this.sprite.play(animKey, true);
      }

  destroy():
    - this.sprite.destroy()
    - this.nameLabel.destroy()
